# 选手数据个人展示页设计

## 目标

把公开页「数据榜」从纯表格改成可切换选手的个人展示页。页面第一眼展示选手身份和近期状态，继续向下保留高密度数据查询能力。

## 形态

- 公开页「数据榜」Tab 内直接展示，不新增独立入口作为主路径。
- 左侧为选手选择器；中小屏改为横向选择器，避免首屏被列表占满。
- 默认选中有数据选手中的 KDA 最高者；无数据时展示空态。
- 个人区采用混合版：主页头图 + 红绿最近走势条 + 高密度数据明细。

## 数据

后端统计继续基于当前活跃赛事，只计入非草稿局。

个人统计需要返回：

- 胜率：`wins / games`，无场次时为 0。
- 场均 K/D/A、KDA、场均补刀、伤害、金币、MVP 数。
- 常用英雄：按使用场次降序，包含场次、胜场、胜率、KDA、场均伤害。
- 最近比赛：按比赛时间倒序，用于红绿走势条和明细表。

榜单接口需要提供足够字段让前端在不逐个请求选手详情的情况下完成切换。单选手 API 保留，用于比赛详情页的跳转兼容。

## UI

头图显示：

- 头像占位：使用昵称首字母。
- 昵称、队伍、位置、代表英雄、最近状态。
- 核心指标：胜率、KDA、MVP。
- 最近 8 场红绿走势条，W/L 颜色表达胜负。

详情区显示：

- 核心数据卡：场次、场均 K/D/A、场均伤害、场均金币、场均补刀。
- 常用英雄列表。
- 最近比赛表格，保留比赛、对手、英雄、K/D/A、伤害、金币、MVP，并提供明确的比赛详情入口。

## 验证

- 服务测试覆盖胜率、常用英雄聚合、最近比赛倒序、跨赛事隔离。
- 组件测试覆盖默认选中、切换选手、红绿走势、搜索过滤。
- 运行相关 vitest 和 typecheck。
