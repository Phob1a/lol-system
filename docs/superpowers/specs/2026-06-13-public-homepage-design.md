# 公开首页入口设计

日期：2026-06-13  
状态：rev.1  
前置：赛季报名、选秀直播、tournament M1/M2、赛程时间表已完成

## 1. 背景

当前 `/` 由 `src/app/page.tsx` 重定向到 `/login`，`src/middleware.ts` 又在用户已登录时把 `/` 按角色跳到 `/admin` 或 `/captain`。这在系统只有后台和队长端时成立；现在公开功能已经扩展到报名、赛事赛程、数据榜、比赛详情、选秀直播，根路径继续展示登录模块会误导公开用户。

目标是把 `/` 改为公开门户首页，让第一次访问系统的人能直接找到开放功能；登录仍保留为管理入口，而不是首页主体。

## 2. 范围

本期做轻量门户，不做营销官网。

包含：
- `/` 展示公开入口：首页卡片/按钮指向报名、赛事赛程、数据榜、选秀直播、登录。
- 首页显示当前赛季的基础状态，用于解释入口是否可用。
- `/login` 保留为独立登录页。
- middleware 不再把 `/` 当作受保护入口，也不再对已登录用户访问 `/` 自动跳转后台。

不包含：
- 新增数据库字段。
- 改造 `/tournament` 的内部 tab 路由能力。
- 首页倒计时、直播状态实时轮询、复杂 CMS 配置。
- 管理后台信息暴露到公开页。

## 3. 页面行为

### 3.1 首页结构

`/` 使用服务端页面，直接读取当前活跃赛季和公开赛事状态。页面布局保持工具型、扫描友好：

- 顶部：系统名、当前赛季名、赛季状态文案。
- 主入口区：
  - 报名：跳 `/register`。仅当赛季状态为 `REGISTRATION` 时强调为主入口；未开放时仍可进入并看到现有“报名未开放”说明。
  - 赛事赛程：跳 `/tournament`。
  - 选手数据榜：跳 `/tournament`，文案注明在赛事页“数据榜”中查看。当前 `PublicTournamentView` 还没有 hash/tab 直达能力，本期不强行新增。
  - 选秀直播：跳 `/live`。
  - 登录后台：跳 `/login`。
- 状态区：用简短文案展示报名状态、赛事状态、选秀入口状态。没有活跃赛季时展示“暂无开放赛季”，仍保留登录入口。

### 3.2 入口排序

入口顺序按赛季状态调整，赛事状态只影响辅助文案，入口集合保持稳定：

- `REGISTRATION`：报名优先，其次赛事赛程、选秀直播、登录。
- `ROSTER_LOCKED`：赛事赛程优先，其次选秀直播、数据榜、登录；报名入口降权并显示“报名已截止”。
- `DRAFTING`：选秀直播优先，其次赛事赛程、数据榜、登录。
- `COMPLETED`：赛事赛程和数据榜优先，其次直播、登录。
- `SETUP`：显示“赛季准备中”，保留赛事赛程、报名、登录入口。
- 无活跃赛季：显示空态，保留登录入口。

状态判断只用于展示优先级和文案，不作为权限边界。真正的可访问性继续由各目标页面和 API 自己决定。

## 4. 路由与认证

### 4.1 `src/app/page.tsx`

移除 `redirect('/login')`，改为渲染公开首页。首页不要求登录，也不根据登录角色自动跳转。

原因：公开首页的核心用户是未登录访问者；已登录管理员或队长也可能需要查看公开展示效果，强制跳转会妨碍这个用途。

### 4.2 `src/middleware.ts`

把 `/` 纳入公开路径：

- 未登录访问 `/`：允许。
- 已登录访问 `/`：允许。
- `/admin/**`、`/captain/**`、`/change-password` 等受保护路径保持现有校验。
- `/login` 继续公开；登录成功后的角色跳转仍由登录流程处理。

需要删除或绕过当前 `pathname === '/'` 的角色跳转分支，否则 `src/app/page.tsx` 的公开首页对已登录用户不可达。

## 5. 数据来源

首页只读取已有公开或低敏数据：

- 活跃赛季：复用 `getActiveSeason(prisma)`。
- 赛事是否存在/状态：可以复用 tournament 公开读模型，或直接查询当前赛季下 tournament 的最小字段。

不得在首页暴露：

- tournament config/version。
- 管理端状态、报名详细名单、队伍账号、选手费用明细。
- 草稿局或管理端录入数据。

## 6. UI 约束

- 不做大面积营销 hero；第一屏直接呈现可用入口。
- 卡片只用于入口项，不嵌套卡片。
- 移动端入口单列，桌面端 2-3 列。
- 首页文案只解释当前状态，不写使用教程。
- 登录入口存在但视觉权重低于当前公开主任务。

## 7. 测试

最低覆盖：

- middleware 单元/集成可行处验证 `/` 不重定向 `/login`，未登录和已登录均可进入。
- 首页无活跃赛季渲染空态和登录入口。
- `REGISTRATION` 赛季首页突出报名入口。
- 存在赛事时首页展示赛事/赛程入口。
- `npm run typecheck` 与相关测试通过。

如仓库缺少 middleware 测试基建，可先覆盖页面渲染与入口 href，middleware 行为在 E2E 或手动 curl 中验证。

## 8. 验收标准

- 打开 `http://101.132.96.101/` 首屏是公开门户，不是登录页。
- 点击报名、赛事、直播、登录入口都能到达现有页面。
- 未登录用户访问 `/admin` 仍会跳 `/login`。
- 已登录管理员/队长访问 `/` 不被强制跳走。
- `/login` 仍可直接访问并完成原有登录流程。
