# 队伍总费用并入赛事管理设计

日期：2026-06-15 ｜ 状态：implemented ｜ 前置：赛事物理合一已上线，手动淘汰赛排位已上线

## 1. 背景

当前后台左侧有独立「系统配置」入口，页面只承载当前赛事的「队伍总费用」设置。这个配置不是全局系统参数，而是当前赛事的规则参数；继续单独放一个菜单会让后台信息架构变碎，也容易让管理员误以为它跨赛事生效。

用户已确认收敛方案：删除独立「系统配置」菜单，把队伍总费用并入「赛事管理」当前赛事设置里。

## 2. 目标与非目标

| 项 | 结论 |
|---|---|
| 目标 | 在「赛事管理」中编辑当前赛事的队伍总费用；左侧不再展示「系统配置」 |
| 目标 | 保留现有锁定语义：选秀开始后队伍总费用不能修改 |
| 目标 | 保留现有更新接口与校验语义，不改数据模型 |
| 非目标 | 不移动选手个人费用；选手费用仍属于报名/名册属性，继续留在「报名管理」 |
| 非目标 | 不重做后台整体导航，不改赛事赛制配置流程 |

## 3. 用户流程

### 3.1 有当前赛事

1. 管理员进入「赛事管理」。
2. 在当前赛事的「设置」页签中看到「队伍总费用」卡片。
3. 在允许阶段内修改费用并保存。
4. 保存成功后刷新赛事状态并提示「队伍预算已更新」。

### 3.2 无当前赛事

无当前赛事时，「赛事管理」仍展示创建赛事表单。队伍总费用设置不单独出现，因为它依赖具体赛事。

### 3.3 已锁定阶段

赛事进入选秀开始后相关阶段，队伍总费用输入框和保存按钮禁用，并展示锁定提示。原因保持不变：各队剩余预算已基于队伍总费用计算，继续修改会造成预算快照漂移。

## 4. 实现设计

### 4.1 UI 收敛

删除左侧导航中的「系统配置」入口。

「赛事管理」的 `SetupTab` 增加一个队伍总费用区块：

- 展示当前 `tournament.teamBudget`。
- 复用现有 `BUDGET_EDITABLE_STATUSES` 判断是否可编辑。
- 保存时调用现有 `PATCH /api/tournament/[id]`，body 为 `{ teamBudget: number }`。
- 成功后调用 `refetch()`，保持赛事管理页内状态同步。

现有 `SeasonConfig` 组件和 `/admin/config` 页面可以删除；若为了兼容短期保留路由，也不再从主导航暴露。推荐直接删除，避免继续保留「Season」命名残留。

### 4.2 数据与接口

不新增表、不改 Prisma schema、不新增迁移。

沿用现有 `PATCH /api/tournament/[id]` 更新队伍总费用。后端当前已具备：

- 费用必须为正数。
- 仅 `BUDGET_EDITABLE_STATUSES` 内状态允许修改。
- 选秀开始后拒绝修改，防止预算快照漂移。

### 4.3 测试

需要覆盖：

- 左侧导航不再出现「系统配置」。
- `SetupTab` 有当前赛事时展示队伍总费用输入。
- 可编辑阶段修改费用会调用 `PATCH /api/tournament/[id]`，成功后 `refetch()`。
- 锁定阶段输入和保存按钮禁用，并展示锁定提示。
- 报名管理中的选手个人费用编辑不受影响。

## 5. 部署影响

这是纯前端/API 复用改动，无 schema 变更，无数据迁移，无需 seed。上线流程按普通代码发布：build → restart → smoke。

## 6. 验收标准

- 后台左侧不再有「系统配置」。
- 管理员可在「赛事管理」当前赛事设置中调整队伍总费用。
- 选秀开始后队伍总费用不可编辑。
- 选手个人费用仍在「报名管理」中编辑。
